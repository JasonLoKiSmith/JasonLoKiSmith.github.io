2. Web Applications

TL;DR:

  • Me on MVC - EXCELLENT
  • Me on MVVM Concepts - EXCELLENT
  • Me on Angular - Meh (As of Feb 2020)
  • Me on React - No exposure (As of Feb 2020)
  • Me on TypeScript - I would marry it if I could

I have been doing web development since .NET 3.5 was in its infancy. The only way to build websites then was using ASP.NET Webforms and having to manage that dreadful ViewState object. Over the years I have worked with various other web technologies, but the main one that stuck with me must have been MVC. I absolutely love MVC and how dynamic it is and the possibilities it brings to the web development world.

MVVM

I love researching different tools in programming and to be honest, in today's world it is necessary to do it in order to keep up with trends. I have used the familiar MVVM pattern of building responsive front ends when I was exposed to WPF for a few years and I was wondering if there is something similar in the web space. To my amazement I found out this has been a long standing trend! I then went and looked at a common and well supported library to start learning these MVVM bindings and patterns. Thats when I met Knockout.js. It was fairly easy to get used to and I have managed to build some awesomely responsive Web front-ends using it.

Angular + React

Angular and React have become so widely popular its unbelievable. With this in mind, it was inevitable for me to climb on either the angular or the react train.

I will be honest, if I had to rate myself on my knowledge on angular it would still be novice at best. As of February of 2020, I have been exposed to angular for roughly 8 to 9 months. The main issue with regards to why I rate myself so low is I have always been tasked with making changes to existing angular applications. I have never built an angular application from scratch which is, where I feel, the most hard-core exposure to a technology comes from. Having extensive experience in the MVVM way of doing things (and things like JQuery Ajax etc), the learning curve of doing angular bindings was fairly easy.

TypeScript

Coming from a C# world where everything is neatly structured and strongly typed it did not take me long to fall absolutely in love with TypeScript. For me, TypeScript reads a lot like those cheesy love stories my wife loves so much to read, to her the stories are always so sweet even though the endings and story lines are pretty predictable. I honestly wish TypeScript will become natively supported by modern browser
